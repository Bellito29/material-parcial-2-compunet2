Ataque principal: si solo quieres “una lista grande” para memorizar, estás estudiando mal. Eso sirve para reconocer patrones, pero no para resolver cuando te cambien una relación o un campo. Aun así, te la doy porque te puede servir como banco de práctica.

Te voy a dar muchas combinaciones **tipo parcial**, usando el estilo de Spring Data JPA derivado por nombre de método. Las agrupo por dificultad para que no mezcles lo básico con lo que ya empieza a oler a `@Query`.

# 12. Consultas trampa que parecen válidas, pero debes pensarlas

Estas pueden compilar conceptualmente, pero a veces ya piden `@Query` o se vuelven demasiado feas:

```java
findDistinctByHomeCountry_Players_PlayerClubs_Club_NameAndHomeCountry_Players_Position(String clubName, String position)
findDistinctByAwayCountry_Players_PlayerClubs_EndDateIsNullAndAwayCountry_Players_FifaScoreGreaterThan(Integer score)
findDistinctByPlayerClubs_Player_Country_HomeMatches_StadiumAndPlayerClubs_Player_Country_HomeMatches_MatchDateBetween(
    String stadium, LocalDate start, LocalDate end
)
```

Estas te las pongo no para que las memorices, sino para que entiendas cuándo el nombre empieza a ponerse ridículo y ya deberías pensar en `@Query`.
